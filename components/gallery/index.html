Galleria — Documentación del Componente (PrimeVue v4)

Componente de galería multimedia con soporte para miniaturas, navegación, pantalla completa y responsive.

Import

import Galleria from 'primevue/galleria';

Uso Básico

Requiere un value (array de objetos), #item para la imagen principal y #thumbnail para las miniaturas.

<Galleria :value="images" :numVisible="5">
  <template #item="slotProps">
    <img :src="slotProps.item.itemImageSrc"
      :alt="slotProps.item.alt"
      style="width: 100%; display: block" />
  </template>
  <template #thumbnail="slotProps">
    <img :src="slotProps.item.thumbnailImageSrc"
      :alt="slotProps.item.alt"
      style="display: block" />
  </template>
</Galleria>

Fullscreen

Se activa con :fullScreen="true" y v-model:visible.

<Galleria v-model:visible="visible"
  :value="images" :fullScreen="true"
  :showItemNavigators="true"
  :showThumbnails="true" :numVisible="9">
  <template #item="slotProps">
    <img :src="slotProps.item.itemImageSrc"
      style="width: 100%; display: block" />
  </template>
  <template #thumbnail="slotProps">
    <img :src="slotProps.item.thumbnailImageSrc"
      style="display: block" />
  </template>
</Galleria>

Responsive

Configuración por breakpoint vía responsiveOptions.

const responsiveOptions = [
  { breakpoint: '1920px', numVisible: 5 },
  { breakpoint: '1440px', numVisible: 5 },
  { breakpoint: '1280px', numVisible: 4 }
];

Accesibilidad

Basado en la documentación oficial de PrimeVue Galleria.

Screen Reader

El componente usa role="region" como contenedor principal. aria-label y aria-roledescription se pueden añadir para describir el carrusel. El contenedor de slides tiene aria-live="polite" (o "off" en autoplay). Cada slide tiene role="group" con aria-roledescription="slide". Los slides inactivos se ocultan con aria-hidden.

Los botones de navegación tienen aria-label que se configura vía aria.prevPageLabel y aria.nextPageLabel en la API de locale. Las miniaturas siguen el patrón de tabs: role="tablist" / role="tab" con aria-selected y aria-controls.

En fullscreen, el overlay usa role="dialog" con aria-modal="true". El botón de cierre usa aria-label desde aria.close del locale.
